Manejo de Errores en Python

Evitar caídas y mejorar la experiencia del usuario

El manejo de errores permite que un programa continúe funcionando de manera controlada ante situaciones inesperadas, evitando “crashes” y proporcionando mensajes útiles al usuario. Python utiliza principalmente los bloques try, except y finally para capturar y gestionar excepciones.

Estructura Básica

  • try: Contiene el código que puede generar un error.
  • except: Captura y maneja el error específico.
  • finally: Se ejecuta siempre, ocurra o no el error (ideal para limpieza). 
try:
    # Código que puede fallar
    pass
except TipoDeError:
    # Manejo del error
    pass
finally:
    # Código que siempre se ejecuta
    pass

⚠️ Errores comunes y su manejo

1. ValueError

Valor Inapropiado

📌 ¿Qué es? Ocurre cuando una función recibe un valor del tipo correcto pero inapropiado (ej. intentar convertir "abc" a entero).

Ejemplo típico (Falla):

# Si el usuario escribe "abc" crashea
edad = int(input("Ingresa tu edad: "))

Manejo adecuado:

try:
    edad = int(input("Ingresa edad: "))
    print(f"Tienes {edad} años")
except ValueError:
    print("Error: Ingresa un número.")
Buenas prácticas:
  • Validar inputs antes de convertirlos.
  • Dar mensajes claros: evitar mensajes técnicos confusos para el usuario.
2. FileNotFoundError

Archivo no encontrado

📌 ¿Qué es? Se produce cuando se intenta abrir un archivo que no existe en la ruta especificada.

Ejemplo típico (Falla):

# Si datos.txt no existe, crashea
archivo = open("datos.txt", "r")

Manejo adecuado:

try:
    archivo = open("datos.txt", "r")
except FileNotFoundError:
    print("El archivo no fue encontrado.")
finally:
    print("Operación finalizada.")
Buenas prácticas:
  • Verificar existencia del archivo antes (os.path.exists).
  • Ofrecer alternativas (crear archivo automáticamente, pedir nueva ruta).
3. ZeroDivisionError

División por cero

📌 ¿Qué es? Ocurre matemáticamente cuando se intenta dividir cualquier número entre cero.

Ejemplo típico (Falla):

# Error fatal en ejecución
resultado = 10 / 0

Manejo adecuado:

try:
    n1 = float(input("Núm 1: "))
    n2 = float(input("Núm 2: "))
    print(f"Resultado: {n1 / n2}")
except ZeroDivisionError:
    print("Error: No dividas por 0.")
except ValueError:
    print("Error: Solo números.")
Buenas prácticas:
  • Validar el divisor (if n2 == 0) antes de la operación si es posible.
  • Prevenir el error con lógica de negocio en lugar de solo capturarlo.

🧩 El poder de finally

El bloque finally se ejecuta siempre, incluso si ocurre un error o si la función hace un return previo. 

try:
    archivo = open("datos.txt", "r")
    # ... operaciones ...
except FileNotFoundError:
    print("No encontrado.")
finally:
    print("Cerrando proceso...")
    # archivo.close() iría aquí
Ideal para:
  • Cerrar archivos y conexiones de base de datos.
  • Liberar recursos de memoria.
  • Registrar logs de finalización.

Estrategias Anti-Crashes

  • Validación previa: Verificar inputs, comprobar archivos y validar divisiones antes de ejecutar.
  • Manejo específico: Evita usar except: genéricos. Es mejor especificar except ValueError:.

Mensajes útiles al usuario

Un buen sistema no solo evita fallar, también comunica claramente qué ocurrió, por qué y cómo solucionarlo.

"Error 404 / Traceback exception"
"El archivo no existe. Verifica el nombre o la ubicación."
